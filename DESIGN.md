# DESIGN.md — 设计规范

## 产品与设计基调

人工智能训练师五级零基础练习与考试系统。用户：职业培训学校零基础学员（部分几乎没用过电脑）、教师、管理员。场景：培训机房 PC（1366×768）为主。

**意象锚点**：培训教室里的护眼绿黑板 + 干净白色练习册 + 老师的大字板书。耐心、稳重、可信赖，像一位不急不躁的老师。

## 气质与意象

- 关键词：耐心、清晰、稳重、鼓励、严肃（考试）
- 场景感：上午的培训教室，阳光照在白色练习册上，黑板是护眼的深绿色，板书字大而清晰；学员每一步操作后都能得到一句"做对了"的确认。

## 视觉策略

- 大字号、大按钮、大点击区域；每屏只突出一个主要任务。
- 图标必须配中文文字，禁止纯图标按钮表达关键操作。
- 状态绝不只用颜色表达：成功=✓图标+"做对了"文字；错误=✗图标+"哪里错、怎么改"文字。
- 练习模式：温和鼓励；考试模式：克制严肃（去掉提示色块与装饰）。

## 配色方案

- 主色 primary：墨青绿 `oklch(0.45 0.09 175)` —— 来自教室护眼黑板绿，传递稳重与"可以通过"的信心。
- 背景 background：暖白纸 `oklch(0.985 0.004 95)`；学习阅读区用极浅护眼绿 `oklch(0.96 0.02 155)`。
- 强调 accent（重要操作/鼓励）：暖橙 `oklch(0.68 0.15 60)`，如老师批改作业的暖色笔迹。
- 成功 success：青绿（与主色同族，靠图标+文字区分）；警告 warning：琥珀；错误 destructive：沉稳砖红（不刺眼）。
- 边框与分隔：低对比暖灰，避免生硬纯黑线。

## 字体排版

- 字体族：系统中文字体栈（PingFang SC / Microsoft YaHei / Noto Sans SC 回退），机房机器必可达，不依赖网络字体。
- 基准字号：全局 16px；学员实操页正文 18px；任务目标一句话 20px 加粗；主按钮文字 18px。
- 标题：粗黑体感（font-weight 700），层级靠字号与间距而非颜色。

## 动效与交互

- 性格：安静、功能性。仅用于状态确认（打勾淡入、保存成功提示）与引导（当前步骤高亮呼吸）。
- 时长 ≤200ms，ease-out；无滚动视差、无页面转场动画。
- 悬停：清晰反馈（加深/描边），焦点态必须有可见 focus ring（键盘可用）。

## 布局与响应式

- 学员实操：单栏任务流，顶部一句话任务目标 + 大操作区 + 底部固定主操作栏。
- 管理/教师端：侧边导航 + 表格密集布局，但字号仍 ≥14px。
- 断点：1366×768 完整可用；移动端可查看成绩/进度，实操考试 PC 优先。

## 设计禁忌

- 禁止科技蓝+蓝紫渐变、玻璃拟态、暗色优先。
- 禁止小于 14px 的正文；禁止纯图标关键按钮；禁止颜色作为唯一状态信号。
- 禁止专业术语直出（如把 IoU 显示给学员）；错误提示必须说"哪里错、怎么改"。
- 禁止隐藏手势与长按等易误触交互；禁用花哨光标。
